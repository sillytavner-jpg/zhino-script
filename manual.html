AI 聊天记忆管家

明月秋青智脑

你在酒馆和角色聊天，它在后台默默帮你 记、理、用。 所有的角色记忆、剧情时间线、关系变化，它都打理好了，你只管聊。

数据全在你电脑上

记

每轮对话后自动总结刚才发生了什么、出现了什么新物品和新地点。聊久了还会做一次"大总结"，把整段剧情整理成清晰的时间线，写入每个角色的记忆档案。

理

分析角色之间的关系变化、每个角色的性格随着剧情怎么演变、你的互动风格有什么特点。还会把故事里出现过的地点和物品串成一个知识网，后台推演不在场角色的独立行动。

用

每次 AI 要回复你的时候，智脑从记忆库里挑出最相关的内容，悄悄塞进上下文。所以角色聊着聊着就会"记得"你们之前一起经历过的事。

安装

导入一行脚本，30 秒搞定。

1 打开酒馆 → 顶部菜单"扩展管理" → "酒馆助手" → "脚本" → "新建脚本" → "全局脚本"
2 在输入框中粘贴下方地址
3 点击确认。稍等几秒，出现紫色的悬浮窗即安装成功
→ import 'https://cdn.jsdelivr.net/gh/sillytavner-jpg/zhino-script@v5.0.2/dist/index.js'
更新版本：扩展管理 → 酒馆助手 → 脚本，找到智脑脚本，点编辑，把地址里的 @v5.0.2 换成最新版本号即可。

五分钟配置

装好之后做三件事：配 API、开向量召回、填一份你的人设。之后就只管聊。

第一步：配置 API

打开智脑面板 → 设置 → 网络与 API → 管理 API，在这里添加你的 API。智脑的后台任务需要调用模型来生成总结和记忆，推荐按以下分配：

任务类型推荐模型说明
大总结 / 角色记忆Gemini 3.1 Pro 或 DeepSeek V4 Pro需要强理解力，不要用小模型
小总结 / 世界推进Gemini 3 Flash 或 DeepSeek V4 Fast轻量任务，快速模型即可
其余全部Gemini 3.1 Pro动态人设、梦呓、关系分析等
调度模式：在管理 API 页面的"调度模式"中，如果你的渠道支持同时 5 个并发调用，或者你给不同功能分别配了不同渠道的 API，建议开启并发模式——多个任务同时跑，体验更流畅。否则选排队模式，任务按顺序逐个执行。

第二步：开启语义召回

设置 → 网络与 API → 开启"语义召回"。推荐使用硅基流动（SiliconFlow）的免费向量模型 BAAI/bge-m3：

  • 去 siliconflow.cn 注册，在 API 密钥页面复制你的 Key
  • 回到智脑召回设置，粘贴 Key，模型填 BAAI/bge-m3
  • 地址默认即可（https://api.siliconflow.cn/v1）
  • 填好后点一下测试连接，显示成功就配置完了

语义召回能让记忆匹配从"关键词"升级为"含义理解"——比如能认出"打架"和"起冲突"是同一件事。长篇多角色时效果显著，短篇可以先不开。

第三步：填写你的用户人设

1 打开智脑面板，点击 玩家 标签，进入 用户人设
2 点击 新建，用几句话描述你自己——性格、说话风格、习惯都行，不用写很长
3 点击 保存并分析，AI 自动生成你的互动画像。分析完成后角色就能更懂你的风格
完成后：先正常聊几十轮，再打开智脑面板看看。角色记忆、动态人设、剧情时间线都已经自动生成了。之后只需根据自己的习惯微调设置即可。

重点功能详解

深入了解五个核心模块的运作机制。

梦呓 — 行为翻译手册

"AIRP 就像一场梦，梦中的呓语。"梦呓不是性格总结，而是一份"行为翻译手册"——告诉 AI 当你做了某个动作时，应该理解成什么、不要误解成什么。

分析内容

  • 基础信息：姓名/性别/年龄/身份，从对话中能推断的部分
  • 性格调色盘：底色、主色调、当前压力值——AI 用来理解你当下的情绪基调
  • 肢体接触习惯：对不同角色的接触方式、频率、触发条件
  • 说话方式：句式特征、语气偏好、典型回应模式
  • 情绪表达：每种情绪 → 你的外在表现 → AI 禁止误读成什么
  • 按角色分组：你对每个角色的独特互动模式
  • Roll 偏好：你倾向于哪种叙事风格

运作机制

  • 触发：每 10 轮 AI 回复后独立触发，不依赖大总结
  • 分析来源：聊天中你所有的用户输入记录 + 你填写的用户人设
  • 注入位置：用户消息中、AI 正文生成之前——让 AI 在写回复前先"读懂你"
  • 注入策略：按当前在场角色条件注入，不在场的角色交互模式会被跳过
  • 可编辑：面板中所有字段可手动修改，支持撤回/恢复
  • 防污染：NSFW 梦呓单独存储，只在标记为 NSFW 的对话中注入

知识图谱 — 地点 + 物品 + 人物关系网

故事世界的结构化映射。不用单独调用 API，搭车小总结顺带维护。

类型节点边说明
地点 名称、简述、别名、语义向量 contains（包含子地点）
connected（连通 + 路径描述）		 每轮小总结自动发现新地点、更新位置关系
物品 名称、简述、数量、别名、是否已消耗 belongs_to（归属角色/地点） 消耗标记后不再作为当前可用物品注入，避免"已经喝掉的药水"反复出现
人物 名称、当前位置、别名 位置映射（不在边表，单独记录） 跟踪每个角色当前所在地点，世界推进和场景注入都依赖它

关键机制

  • 搭车更新：小总结的 AI 调用同时输出 graphDiff（add/update/delete），智脑按 id 去重合并进主图。一趟 API 两份产物。
  • 版本快照：每次图谱变更记录版本号+对应楼层。最多保留 6 张图谱快照，超出后自动淘汰最旧版本。重 roll 某楼层后，该楼层及之后的图谱版本被自动截断，用楼层前状态重建——因此最多支持退回 5 轮对话。
  • 语义向量：地点和物品可选生成 embedding 向量，用于召回时按含义匹配（而非仅按名称）。
  • 上下文注入：每次 AI 回复前，按 6 层优先级注入图谱：玩家可用物品 → 同场角色物品 → 当前地点物品 → 当前地点+相连地点 → 玩家输入提及地点物品 → 正文相关召回。
  • SVG 可视化：面板中可交互浏览，地点圆形节点 + 角色小圆分布周围，四种连线颜色区分关系类型。

世界推进 — 不在场角色的独立生命

防止故事世界"冻结"——主角在 A 地推进剧情时，B 地的角色也有自己的生活。

前置条件：世界推进必须与以下模块同时开启才能正常工作：① 小总结（提供图谱数据和在场角色判定）；② 角色记忆（提供角色的既往事实和态度）；③ 知识图谱自动（提供角色位置、物品、连通路径）。此外，推演前需在「世界」面板中手动选择绑定的世界书条目——条目中应包含不在场角色的设定信息，否则 AI 推演缺乏角色依据。

🔄 三步联动闭环

第一步 · 输入 — 知识图谱 + 角色记忆双源取料

智脑调用 selectWorldProgressCandidates() 自动筛选不在场角色（当前楼层不在 presentCharacters 中），综合态度、上次互动距今轮数、手动指定等维度打分排序，最多取 2 个。对每个候选角色：

  • 从知识图谱取：当前位置 + 所在地全部物品 + 一跳连通地点路径 + 到主角所在地的最短路线（BFS 搜索）
  • 从角色记忆取：最近 10 条核心/近期记忆，确保推演行动与既往事实、态度、目标连续
  • 可选从世界书取：用户在面板中选择绑定的世界书条目作为额外参考

第二步 · 推演 — AI 精推行动

AI 在 <thinking> 中逐角色执行思维链：确认时间切片 → 检查地点/物品/路径 → 参考记忆确保连续 → 用调色盘判断行动方向。每个角色输出一条第一人称近期记忆（30-80 字），可选附带轻微结果和自然入场引导：

  • level 0：不暴露（entryHint 为 null，不强行引入角色）
  • level 1：痕迹/物品/传闻（路过痕迹、遗留物品、旁人提及）
  • level 2：口信/书信/通讯（可自然插入的远程互动）
  • level 3：短暂本人入场（场景允许时角色短暂出现）

第三步 · 消费 — 三路输出

  • → 注入 AI 上下文：无入场引导的角色动态注入为 <world_state>（"场外角色动态"）；有入场引导的插在用户消息梦呓位置上方
  • → 合并进角色记忆：下一次大总结执行 mergeWorldProgressIntoSummary()，将覆盖楼层范围内的推演记忆按剧情时间排序合并进角色记忆档案，标记 archivedInSummaryVersion
  • → 辅助小总结：最近一条未消费的推演记录作为材料发给小总结，提供"中心角色"和"中心地点"判定依据，消费后标记 smallSummaryConsumed

触发时机：世界推进不在 AI 回复后立即执行，而是标记 pending → 等玩家下一次发消息时才后台推演。避免与 AI 正文生成抢 API、阻塞聊天体验。默认每 2 轮触发一次。

角色记忆 — 核心永久 + 近期更替

每个角色的第一人称记忆档案，分两层管理。

两层记忆结构

  • 核心记忆（永久）：决定角色本质的关键事件和关系。不会被自动删除，只有用户手动降级为近期后才会被自然替换。每条记忆带时间戳和版本号。
  • 近期记忆（自动更替）：最近 N 次大总结范围内的互动记录。每次新总结后旧近期记忆被新生成的内容覆盖。保留最近 3 个版本的近期记忆。

更新与合并

  • 并发执行：与大总结同时调用 API（Promise.allSettled），互不阻塞。任一方失败可独立重试。
  • 世界推进合并：大总结完成时，覆盖楼层范围内的世界推进记忆自动合并进角色记忆，按剧情时间排序，标记 archivedInSummaryVersion 防止重复合并。
  • 追忆面板：按大总结版本分组展示全部历史记忆。可手动编辑、升降级（核心 ↔ 近期）、逐条删除。

生成规格

每条记忆由 AI 以角色第一人称撰写（"我……"），长度控制在 15-40 字。附带激活关键词（子串模糊匹配用）和人物关联列表。每角色每次大总结生成 4-8 条记忆（可调）。AI 被要求反极端化——不写"永远""绝对""彻底"，用"当前""倾向""最近"等柔性词。

记忆召回 — 精准注入相关往事

不是把所有记忆一次塞进来，而是按关系链精准激活——这是角色能"回忆"起久远之事的核心机制。

🎯 神经链架构

召回按关系链组织，锚点是当前场景中的在场角色：

  • 链 1（用户 → 角色）：用户与每个在场角色之间的相关记忆
  • 链 2（角色 → 角色）：在场角色之间的相关记忆
  • 去重：链 2 过滤掉已在链 1 注入的文本，避免同一条记忆重复出现

🔤 关键词匹配（默认路径，无需额外配置）

  • 子串模糊命中：≥4 字的关键词提取 2-3 字滑动窗口，任一子串出现在对话中即命中。解决 AI 生成的复合词（如"腥臭爱液"）在自然对话中只出现片段（"爱液"）的问题。
  • 用户侧反向匹配：用户输入中的连续词块只要被关键词包含即命中。比如用户说了"蛊"，会匹配到触发词"月光蛊"。
  • 打分排序：综合匹配质量（0.50）+ 时间衰减（0.10）+ 关键词密度（0.15）+ 角色重叠度（0.15）+ 事件重要度（0.10），取分高者注入。

🧠 语义召回（可选开启，需配置 embedding API）

  • 用统一查询向量（用户输入 + 最近 AI 回复的前 2000 字）对所有带 embedding 的记忆做余弦相似度计算
  • 低于阈值的直接过滤，通过的按 0.60×相似度 + 0.25×时间衰减 + 0.15×重要度 综合排序
  • 可选开启两阶段重排（Rerank）：粗筛后用专用重排模型精排，进一步提升召回精度
  • 语义失败时自动降级为关键词匹配，不阻塞生成流程

📐 向量维度与模型选择

  • 维度越高 = 占内存越多：每条记忆/事件/图谱节点都会生成一个向量存在本地，维度直接影响存储占用和检索速度。高维度（1024+）语义更精细但存储开销大；低维度（256）检索快、省空间，足够满足角色扮演场景的召回精度。
  • 降维限制：硅基流动（SiliconFlow）的 embedding API 仅支持千问系列模型（如 Qwen3-Embedding）使用 Matryoshka 降维参数。如果用的是 BGE-M3 等其他模型，降维参数会被忽略，始终输出模型原生维度。
  • 推荐配置：知识图谱节点推荐 256 维（节点数量大，低维省空间）；角色记忆和事件推荐 1024 维（语义精度优先）。如果不确定模型是否支持降维，先不填维度留空，跑一次后在召回设置面板查看实际维度再决定。

📅 事件回忆分流

  • 近期事件：最近 N 轮大总结的时间线事件全部无条件注入（默认 2 轮）
  • 远期事件：只有触发词（角色+关键词）在对话中命中时才召回
  • 召回上限：用户匹配优先（userHit），AI 侧匹配补充填充剩余配额，每类最多召回 8-15 条（可调）

注入时机：每次 AI 回复前，在 CHAT_COMPLETION_SETTINGS_READY 事件中执行，与知识图谱注入、动态人设注入同一生命周期。世界推进产生的角色记忆也参与召回——即使该角色从未被大总结覆盖过也能被激活。

常见问题

先打开总览，点一下读取历史楼层让智脑知道有哪些对话。然后用批量总结消化——每批 30 层左右比较合适，楼层特别多的话可以设到每批 50 层。总结完成后，去设置 → 高级功能 → 召回设置，点「为旧数据生成向量」，等进度条跑完就行了。小总结不用补。

先检查 API 设置——有可能是 API 配置的问题，确认 Key 和地址都没填错。如果不是 API 的问题，去设置 → 网络与 API，打开API 监听器，再触发一次分析，查看 API 的请求和回复。如果回复内容正常、也没有截断，带上你的截图和 API 回复找常陆茉子反馈。

直接导出聊天记录即可。智脑的所有数据（角色记忆、时间线、知识图谱、世界推进记录、梦呓、动态人设等）全部存储在酒馆的聊天记录中，跟随聊天一起走。把导出的聊天文件放到新电脑的酒馆中导入，智脑数据会自动恢复，无需额外操作。

它会自动重试最多 3 次（每次间隔越来越长）。如果等了一会儿还不动，点中止然后把每批的楼层数量调小一点再试。

重 roll 单层：完全没问题，什么都不用管。智脑会自动重跑该楼层的小总结并替换旧记录，图谱和角色位置同步更新。

多层退回重 roll：请注意以下限制：

  • 已开启知识图谱：图谱最多保留 6 张快照，只支持五轮内退回。超出五轮的旧图谱已被淘汰，无法恢复。
  • 未开启知识图谱：可以退回多层之前，不受图谱快照限制。
  • 大总结：在总览处可手动撤回，回退到指定版本。撤回后覆盖楼层内的角色记忆会被还原到上一版本。
  • 动态人设：不会自动回退，须在面板中手动删除对应版本。
  • 世界推进：退回楼层范围内的推演记录需手动逐条删除。

不建议。智脑的正文捕获、小总结、大总结、角色记忆是一套完整闭环，和其他记忆插件同时运行会产生冲突和重复注入。如果非要用，可以在设置中关掉小总结、大总结和角色记忆，只保留动态人设、梦呓等不冲突的模块。注意：关掉小总结意味着知识图谱也会停止更新（图谱是搭车小总结的），但已生成的图谱数据不会丢失。

智脑的 API 设置界面可以手动编辑破甲词。打开面板 → 设置 → 网络与 API，在管理 API 的弹窗中切换到「破甲词」标签，可以分别编辑大总结、小总结、梦呓、世界推进、剧情导演等各模块的破甲提示词。如果默认破甲在你的 API 渠道上不生效，在这里自定义即可。

明月秋青预设里会自带一个老版智脑。在酒馆的扩展管理 → 酒馆助手 → 脚本中，找到老版本的智脑脚本，删除或禁用即可，只保留新版。

打开人物志 → 角色库，点编辑角色，在角色右侧点击合并按钮，选择要合并的目标角色，确认即可。智脑已经内置了别名自动归一化，但偶尔 AI 还是会用完全不同的称呼，手动合并就能解决。

三者是联动闭环：输入时，世界推进从知识图谱取角色位置/物品/路径，从角色记忆取既往事实和态度；输出后，推演产生的第一人称记忆会注入上下文（"场外动态"），并在下一次大总结时自动合并进角色记忆档案，同时小总结消费推演材料辅助判定场景。所以开启世界推进前，最好先跑过至少一次大总结（确保有角色记忆和知识图谱数据）。

面板 → 记事本 → 图谱子标签，可以可视化浏览整个图谱并手动增删改节点和边。图谱是搭车小总结自动维护的，重 roll 楼层后旧数据会被自动清理。

打开面板 → 记事本 → 图谱子标签，在可视化界面中点击该角色节点，右侧会弹出详情面板，可手动更改角色的当前位置。位置更新后会立即反映到后续的世界推进推演和上下文注入中。
